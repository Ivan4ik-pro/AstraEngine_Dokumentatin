Класс Logger

core/logger.h · logger.cpp ▶
Готово
Logger - Это глобальный статический класс, который обеспечивает потокобезопасную запись, форматирование и хранение системных сообщений с автоматическим управлением памятью и цветовой индикацией в консоли.
// Инициализация
Logger::Init();

// Использование
Logger::Info("Window created: {}x{}", width, height);
Logger::Warn("Low memory: {} MB", mb);
Logger::Error("SDL failed: {}", SDL_GetError());

// Закрытие
Logger::Shutdown();
💡

SDLContext — это компонент, реализующий паттерн RAII для автоматического управления жизненным циклом библиотеки SDL3, гарантирующий её корректную инициализацию и очистку.
// При создании объекта SDL запускается автоматически
SDLContext context;

PUBLIC: SDLContext() (Конструктор) Инициализирует подсистему видео SDL3 и выбрасывает исключение с записью в лог, если запуск завершился ошибкой. ~SDLContext() (Деструктор) Завершает работу SDL3 и записывает в лог сообщение об успешном закрытии библиотеки. Готово
Класс Instance — это удобная обертка над базовым объектом Vulkan (VkInstance), служащая главной точкой входа в API. Он берет на себя всю рутинную работу по инициализации библиотеки, подключению необходимых расширений (в том числе для интеграции с окном SDL3) и настройке системы отладки (слоев валидации).

Класс Device

graphics/vulkan/device.h · device.cpp ▶
Готово
Класс Device — отвечает за поиск и выбор наиболее подходящей видеокарты в системе (Physical Device), а затем создает программный интерфейс (Logical Device) для отправки ей команд. Он настраивает все современные функции GPU (такие как динамический рендеринг и индексация дескрипторов из Vulkan 1.3/1.4) и подготавливает очереди команд для отрисовки графики и вывода её на экран.
💡
